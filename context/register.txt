Genera el código para una pantalla de 'Registro de Nuevo Cliente' en un proyecto Flutter ('foryou'), aplicando Clean Architecture y la siguiente pila tecnológica:

* **Supabase:** Backend principal (Autenticación y Base de Datos Remota).
* **Drift:** Base de datos local (caché).
* **Dio:** Cliente HTTP (para uso general de API REST).

La pantalla será un formulario para registrar un `Cliente`, basado en la estructura de la tabla `clientes` del schema SQL proporcionado.

**Asunción Crítica de Arquitectura:**
Para integrar Supabase Auth, la tabla `clientes` en la *base de datos de Supabase* debe usar el `id` (UUID) de `auth.users` como su Primary Key (`id_cliente`), en lugar de un `INT AUTO_INCREMENT`.

**1. Capa de Datos (Data Layer)**

* **Drift (Base de Datos Local):**
    * Define la base de datos de Drift en `lib/core/data/local/app_database.dart`.
    * Define la tabla `Clientes` (en `app_database.dart`) mimetizando el SQL.
    * **Importante:** `id_cliente` debe ser `TextColumn()` para almacenar el UUID de Supabase. Los campos deben ser: `id_cliente`, `nombre`, `usuario`, `telefono`, `domicilio`, `correo`. **No almacenes el password_hash en Drift.**
* **Datasource Remoto (Supabase):**
    * Crea `ClientRemoteDataSource` en `lib/features/register/data/datasources/client_remote_datasource.dart`.
    * Debe tener un método: `Future<ClientModel> registerClient(RegisterParams params)`.
    * Este método debe:
        1.  Llamar a `supabase.auth.signUp(email: params.email, password: params.password)`.
        2.  Si es exitoso, obtener el `user.id` de Supabase Auth.
        3.  Llamar a `supabase.from('clientes').insert({...})` para guardar los datos *adicionales* (`id_cliente: user.id`, `nombre: params.nombre`, `usuario: params.usuario`, etc.).
        4.  Hacer un `select()` de los datos recién insertados y retornarlos como `ClientModel`.
* **Datasource Local (Drift):**
    * Crea `ClientLocalDataSource` (`.../client_local_datasource.dart`) con un método `Future<void> cacheClient(ClientModel client)` que guarde el cliente en la tabla `Clientes` de Drift.
* **Repositorio:**
    * Crea `ClientRepositoryImpl` (`.../data/repositories/client_repository_impl.dart`).
    * En el método `registerClient`, debe llamar primero al `RemoteDataSource`.
    * Si el `RemoteDataSource` tiene éxito, debe llamar al `LocalDataSource` para guardar el cliente en el caché de Drift antes de retornar el resultado.
* **Modelo:**
    * Crea `ClientModel` (`.../data/models/client_model.dart`) que extienda la entidad `Client` y tenga los métodos `fromJson/toJson`.

**2. Capa de Dominio (Domain Layer)**

* **Entidad:** Crea la entidad `Client` (`.../domain/entities/client.dart`) con los campos (`id_cliente`, `nombre`, `usuario`, `correo`, `telefono`, `domicilio`).
* **Repositorio:** Define el contrato `ClientRepository` (abstracto) (`.../domain/repositories/client_repository.dart`).
* **Caso de Uso:**
    * Crea `RegisterClientUseCase` (`.../domain/usecases/register_client_usecase.dart`).
    * Debe depender del `ClientRepository` y exponer un método `call(RegisterParams params)`.
    * Añade validación de negocio (ej. email con formato válido, password con > 8 caracteres) dentro del UseCase antes de llamar al repositorio.

**3. Capa de Presentación (Presentation Layer)**

* **Pantalla (Screen):**
    * Crea `RegisterScreen` (`.../presentation/screens/register_screen.dart`).
    * Debe contener un `Form` con `TextFormField` para: `nombre`, `usuario`, `correo`, `password`, `telefono` y `domicilio`.
* **Gestor de Estado (BLoC):**
    * Crea `RegisterBloc` (`.../presentation/bloc/register_bloc.dart`).
    * Maneja los eventos (`RegisterSubmitted`) y estados (`RegisterInitial`, `RegisterLoading`, `RegisterSuccess`, `RegisterFailure`).
    * Debe llamar al `RegisterClientUseCase` cuando se envía el formulario.

**Contexto de la Base de Datos (Tabla `clientes` para guía):**
```sql
CREATE TABLE `clientes` (
  `id_cliente` int(10) UNSIGNED NOT NULL,
  `nombre` varchar(255) NOT NULL,
  `usuario` varchar(50) DEFAULT NULL,
  `password_hash` varchar(95) NOT NULL,
  `telefono` varchar(20) DEFAULT NULL,
  `domicilio` varchar(255) DEFAULT NULL,
  `correo` varchar(255) DEFAULT NULL
);
